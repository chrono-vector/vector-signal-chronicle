# GOVERNANCE CONTRACT

This document states ecosystem governance principles for vector-signal-chronicle.
It is not L0a plane law.
The Constitution and Supplement 001 in vector-runtime-governance-public remain the controlling L0a authority.
On conflict, this document defers to L0a.

## 1. Mission

The VECTOR ecosystem exists to provide
reproducible,
auditable,
human-governed
observation and execution governance.

Signal Chronicle observes.

Runtime Governance executes.

These responsibilities remain permanently separated.

---

## 2. Core Guarantees

The system guarantees:

• Chronicle First

Every governance decision must be recorded.

• Observe Before Execute

No execution occurs without prior observation.

• Evidence Before Confidence

Confidence cannot exist without observable evidence.

• Confidence Before Decision

Every governance decision must reference an explicit confidence level.

• Human Authority

Human review is the final authority.

• Execution Boundary

Observation never directly authorizes execution.

• Runtime Isolation

External information cannot directly modify runtime behavior.

• Reproducibility

Observer decisions must be reproducible from recorded evidence.

• Auditability

Every governance decision must be traceable.

---

## 3. Non-Goals

VECTOR does NOT guarantee:

Truth determination.

Semantic correctness.

Interpretation of symbolic content.

Prediction of future events.

Automatic execution from external information.

Automatic runtime recovery.

Replacement of human judgment.

---

## 6. External Signal Rule

ExternalActivationSignal records:

may inform governance

may initiate observation

may influence confidence

must never execute runtime actions

must never authorize recovery

must never bypass human review

---

## 7. Runtime Rule

Runtime modifications require independent runtime evidence.

External observations alone are insufficient.

---

## 8. Chronicle Rule

Chronicle is the permanent governance record.

Governance history must remain auditable.

Superseding decisions create new records.

Existing records are never silently rewritten.

---

## 9. Evolution Rule

New signal providers may be added without modifying Runtime Governance.

Examples:

White Rabbit

Research papers

GitHub events

Security advisories

Telemetry

Future providers

---

## 10. Design Philosophy

Signal Chronicle provides situational awareness.

Runtime Governance provides execution authority.

Observation informs governance.

Governance informs execution.

Human authority governs all execution.

## 13. Weekly Observer Review

Weekly Observer Reviews provide a recurring, documentation-only governance process for reviewing accumulated ExternalActivationSignal records. They organize evidence, identify observed relationships and series, and may recommend subsequent Observer Reviews or Confidence updates. They do not authorize execution, runtime modification, or self-healing.

See [observer_reviews/WEEKLY_OBSERVER_REVIEW_POLICY.md](observer_reviews/WEEKLY_OBSERVER_REVIEW_POLICY.md).
